Historic Web Cybersecurity Events

1. The Samy Worm (2005)

Created by MySpace user Samy Kamkar, this worm exploited a cross-site scripting (XSS) vulnerability by injecting malicious JavaScript into an HTML profile. Within just 20 hours, over one million users unknowingly executed the code and added Samy as a friend automatically.
It remains one of the fastest spreading pieces of malicious code in history and serves as a critical example of how user generated HTML content can be weaponized for large scale attacks.

2. The British Airways Data Breach (2018)

Attackers inserted only 22 lines of malicious JavaScript into British Airways' payment page HTML, enabling silent "formjacking" to steal credit card details from roughly 500 000 customers.
This Magecart style attack demonstrated how tiny, invisible changes to HTML can cause massive real world harm, resulting in a £20 million fine for British Airway under GDPR regulations.

3. The Heartbleed Bug (2014) and HTTPS

The Heartbleed Bug was a critical vulnerability in the OpenSSL library (used by ~66% of web servers) that enabled attackers to steal sensitive data like passwords, private keys, and session cookies from encrypted HTTPS connections.
Unlike many web attacks that target HTML/JavaScript, this flaw exploited a memory leak in the "Heartbeat" extension of SSL/TLS, compromising the core encryption that secures nearly all modern web traffic (e.g., banking, email, e-commerce sites). It forced mass password resets across major platforms (Google, Facebook, Yahoo).

4. The Equifax Data Breach (2017)

The Equifax breach stemmed from an unpatched vulnerability in the Apache Struts web framework, a tool widely used to build dynamic HTML web applications. Hackers exploited this flaw to access sensitive data from 147 million people.
The breach exposed names, Social Security numbers, birth dates, and credit card details, highlighting how outdated web server software and unaddressed HTML/backend framework vulnerabilities can lead to catastrophic data loss.

5. The Morris Worm (1988)

Though predating modern HTML, the Morris Worm is a foundational web/network security event: it infected 10% of the internet's then 60,000 computers by exploiting flaws in Unix-based web and mail servers.
Created by Robert Tappan Morris, it was intended as a research experiment but caused widespread outages, spurring the creation of the first computer emergency response team (CERT) and shaping modern web cybersecurity protocols.
